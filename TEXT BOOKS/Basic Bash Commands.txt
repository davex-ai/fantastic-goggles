                                                               Basic Bash Commands (for navigating and managing files):

pwd: Prints the current working directory.
ls: Lists the contents of the current directory. Use ls -la to show hidden files.
cd <directory_name>: Changes the current directory. Use cd .. to go up one level.
mkdir <directory_name>: Creates a new directory.
touch <file_name>: Creates a new empty file.
rm <file_name>: Removes a file. Use rm -r <directory_name> to remove a directory and its contents.
cp <source_file> <destination>: Copies a file.
mv <source_file> <destination>: Moves or renames a file.
cat <file_name>: Displays the content of a file.
$ cd .. :to go to parent directory 
start<file_name>:  is a Windows command that can be executed within Git Bash. This command will open the file using the default application associated with that file type in your Windows operating system.

git init: Initializes a new Git repository in the current directory.
git clone <repository_url>: Clones an existing Git repository from a remote URL to your local machine.
git commit -am "Adding this.html page": move a file/folder fro initialized stage to local repo
git mv <existing file> <new file>: used to rename a file
git add <file_name> or git add .: Stages changes for a specific file or all changes in the current directory for the next commit.
git commit -m "Commit message": Records the staged changes to the repository with a descriptive message.
git status: Shows the current status of your working directory, including untracked, modified, and staged files.
git init -b main: initializes a repo with a new branch
git push: Uploads local commits to a remote repository.
git pull: Downloads and integrates changes from a remote repository into your local branch.

git merge <branch_to_merge>: Merges changes from one branch into the current branch.
git log: Displays the commit history.
